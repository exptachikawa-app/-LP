# 画像が表示されない問題の解決方法

## 🔍 問題の原因

GitHubリポジトリに **`images/`フォルダと画像ファイル**がアップロードされていないため、画像が表示されません。

---

## ✅ 解決方法：画像をGitHubにアップロード

### **STEP 1: 画像をダウンロード**

以下の2つの画像をダウンロードしてください：

#### **画像1: EXPロゴ**
1. ブラウザで以下のURLを開く:
   ```
   https://page.gensparksite.com/v1/base64_upload/9383fc3bb735acef02f1015e754071ff
   ```
2. 画像を右クリック→「名前を付けて画像を保存」
3. ファイル名: `exp-logo.png`
4. 保存場所: デスクトップなど、わかりやすい場所

#### **画像2: 兵働支援員の写真**
1. ブラウザで以下のURLを開く:
   ```
   https://page.gensparksite.com/v1/base64_upload/3fd1d91295eb20f9cbbad6a0b64e8919
   ```
2. 画像を右クリック→「名前を付けて画像を保存」
3. ファイル名: `hyodo-staff.jpg`
4. 保存場所: デスクトップなど、わかりやすい場所

---

### **STEP 2: GitHubに`images`フォルダを作成**

#### **方法A: ブラウザから（最も簡単）**

1. **GitHubリポジトリを開く**
   - https://github.com/exptachikawa-app/-LP

2. **「Add file」→「Create new file」をクリック**

3. **ファイル名に以下を入力**:
   ```
   images/.gitkeep
   ```
   - `images/` と入力すると、自動的にフォルダが作成されます
   - `.gitkeep` は空のフォルダを保持するためのダミーファイルです

4. **「Commit new file」をクリック**

---

### **STEP 3: 画像をアップロード**

1. **リポジトリのトップページに戻る**

2. **`images`フォルダをクリック**

3. **「Add file」→「Upload files」をクリック**

4. **2つの画像をドラッグ&ドロップ**:
   - `exp-logo.png`
   - `hyodo-staff.jpg`

5. **Commit changes**:
   ```
   Commit message: 画像を追加（ロゴと支援員写真）
   ```

6. **「Commit changes」をクリック**

---

### **STEP 4: 確認**

1. **GitHubリポジトリで確認**
   - `images/exp-logo.png` が存在するか
   - `images/hyodo-staff.jpg` が存在するか

2. **1〜2分待つ**
   - GitHub Pagesが自動的に再ビルドされます

3. **公開URLにアクセス**:
   ```
   https://exptachikawa-app.github.io/-LP/
   ```

4. **キャッシュをクリア**:
   - Windows: `Ctrl + Shift + R`
   - Mac: `Cmd + Shift + R`

5. **画像が表示されることを確認**:
   - [ ] ヘッダーのEXPロゴ
   - [ ] 支援員紹介の兵働支援員の写真
   - [ ] フッターのEXPロゴ

---

## 🔧 トラブルシューティング

### **問題1: 画像をダウンロードできない**

**原因**: URLが期限切れの可能性があります。

**解決方法**:
1. 元の画像ファイルを探す
2. または、EXP立川の公式サイト（https://exp.or.jp/）からロゴをダウンロード
3. 支援員の写真は別途用意

---

### **問題2: 画像をアップロードしたのに表示されない**

**確認事項**:
1. ファイル名が正しいか
   - `exp-logo.png`（小文字、ハイフン）
   - `hyodo-staff.jpg`（小文字、ハイフン）
2. `images/` フォルダ内にあるか
3. ファイルサイズが大きすぎないか（推奨: 各1MB以下）

**解決方法**:
1. ブラウザのキャッシュをクリア（Ctrl+Shift+R）
2. 数分待ってから再度アクセス
3. F12で開発者ツールを開き、「Network」タブで画像が404エラーになっていないか確認

---

### **問題3: 画像のファイル形式が合わない**

**確認**:
- EXPロゴ: `.png` 形式
- 支援員写真: `.jpg` 形式

**修正方法**:
画像編集ソフトまたはオンラインツールで形式を変換

## 📸 画像の推奨仕様

### **EXPロゴ**
- 形式: PNG（透過背景推奨）
- サイズ: 横幅 200〜300px
- 容量: 100KB以下

### **支援員写真**
- 形式: JPG
- サイズ: 横幅 400〜600px
- 容量: 500KB以下
- 推奨: 白背景、正面から撮影、スーツ着用
